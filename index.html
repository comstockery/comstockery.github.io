RUFF DRAFT visual 1: Showing the entire album

Intention: Show the entire album at once. For the entire album, the main attribute is the number of tracks. For each track, data attributes include duration, beats per minute, number of samples. Then for each sample, attributes include genre and release year.

Progress: A donut chart with 31 slices (one for each track on the album), sized by their duration. Each has some 'sprinkles' on it. The number of sprinkles per slice represents the number of unique samples per track. Sprinkles are colored based on their genre and placed radially based on their release year.

RUFF DRAFT visual 2: Generations of samples

Intention: Display the multiple generations of music contained within certain Donuts samples.

Progress: Pulled the data for the samples within Mantronix's King of the Beats, the most-used sample on Donuts.

Notes: I'm not sure if this counts as a "visualization" so I need to double-check on that. I want to make this an interactive element with Bootstrap collapse. Other samples I'd explore are Run-DMC's Here We Go (Live at the Funhouse), the Beastie Boys' The New Style, and Jaylib's No Games.

RUFF DRAFT visual 3: Track-by-track donut charts

Intention: Display the samples used within each track on Donuts. Think of each track 'starting' at the 12:00 position, then rotating clockwise you encounter each sample based on its timing. The positions within the 'record grooves' reflect how those samples are used on the Donuts track: structural, surface, and lyric.

Progress: I have the data for all of the samples but need to set up a consistent scheme so that these can be batch-generated in R.

Notes: I think I'm going to have an issue labeling the sampled tracks and artists. I haven't figured out a way to label those without overlapping the data itself. I hope to avoid requiring mouseover for labeling.
