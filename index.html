RUFF DRAFT visual 1: Showing the entire album

Intention: Show the entire album at once. For the entire album, the main attribute is the number of tracks. For each track, data attributes include duration, beats per minute, number of samples. Then for each sample, attributes include genre and release year.

Progress: A donut chart with 31 slices (one for each track on the album), sized by their duration. Each has some 'sprinkles' on it. The number of sprinkles per slice represents the number of unique samples per track. Sprinkles are colored based on their genre and placed radially based on their release year.

Ideally this shows that Donuts uses plenty of soul/funk/disco samples from the late 1960s and early 1970s and hip-hop samples, both from the golden era (late 1980s) and then-contemporary (early 2000s).

RUFF DRAFT visual 4: THE DONUTS!

Intention: Display the timing of samples on each track. Think of each track 'starting' at the 12:00 position, then rotating clockwise you encounter each sample based on its timing. The positions within the 'record grooves' reflect how those samples are used on the Donuts track: structural, surface, and lyric. Each groove represents a unique sample type x sampled track combination. Donuts tracks with more unique type x track combos will have more grooves.

Progress: Got the geoms correctly placed! This required a lot of dplyr work and somehow much more math on top of that. AND I wrote a function to batch-generate and save each plot.

Notes: I've shown the first DOZEN DONUTS. There's no labeling other the track names but I intend to handle that either with geom_richtext or HTML if I have to. (I see geom_richtext struggles with apostrophes!) I will obviously need to add some labeling to make these coherent, but for now it's pretty exciting to get these rendered!
